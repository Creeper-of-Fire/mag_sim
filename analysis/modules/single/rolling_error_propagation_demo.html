滑动平均 — 误差传递说明

tail_statistics_timeseries.py 中 _rolling_avg & _rolling_err 的算法

1. 数据流水线：从原始粒子到滑动平均

数据经过两个阶段的平均，误差在每个阶段都应减小：

阶段 1：跨 run 平均（M 个 run → 均值）
    x̄[j] = (1/M) Σm=1M xm[j]
    均值的标准误差：sem[j] = σrun[j] / √M

↓   M 个独立 run 的平均使误差缩小 √M 倍

阶段 2：滑动平均（N 个时间步 → 滑动均值）
    X̄[i] = (1/N) Σj=starti x̄[j]
    滑动平均的标准误差：(1/N) √(Σ sem[j]²) = (1/(N√M)) √(Σ σrun[j]²)

↓   N 步滑动平均使误差再缩小 √N 倍

两个阶段独立的，所以最终误差 = 阶段2输出，已经包含了阶段1的缩减。

2. 滑动平均后的两种误差

理论误差 σtheory

来源：有限粒子数的泊松计数统计。每个时间步的测量本身有不确定度 σth[j]。

这是测量精度的极限。

传递公式

σtheory[i] = (1/N) √(Σj σth[j]²)

σth[j] = tail_uncertainty[j]，各时间步独立
窗口 N 越大，理论误差越小（∝ 1/√N）

统计误差 σstat

来源：M 个 run 的实际统计涨落。跨 run 均值的标准误差为 σrun/√M。

这是采样精度的极限。

传递公式

σstat[i] = (1/N) √(Σj (σrun[j]/√M)²)
           = (1/(N√M)) √(Σj σrun[j]²)

σrun[j] = tail_excess_std[j]，M 个 run 之间的标准差
窗口 N 越大、run 数 M 越大，统计误差越小

注意：两种误差不应合成。σ_run 中已包含粒子计数的贡献，合成会重复计算。两条带应独立展示、对比解读。

3. 当前代码的问题

当前代码对两种误差使用了同一个函数 _rolling_err，直接传入原始 σ：

# 理论误差——传的是 σth，正确
roll_th = _rolling_err(th_unc * 100, w)

# 统计误差——传的是 σrun，但应该是 σrun/√M
roll_run = _rolling_err(run_std * 100, w) ← 缺少 √M 的缩减

结果：统计误差被高估了 √M 倍（M=4 时高估 2 倍）。
图中"跨 run 散布(传递)"的误差带比实际应该的大得多。

4. 修正方案

统计误差传入前先除以 √M：

# 理论误差：直接传播（不变）
roll_th = _rolling_err(th_unc * 100, w)

# 统计误差：先转成 sem 再传播
roll_run = _rolling_err(run_std * 100 / np.sqrt(M), w)

图例名称改为："理论误差" 和 "统计误差"。
