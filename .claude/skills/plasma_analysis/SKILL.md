## 运行方式

该程序是 Rich 交互式 TUI，有多层嵌套 `Prompt.ask`。通过 **stdin 管道**驱动：

```bash
printf "输入序列" | ~/bin/analysis_python analysis_console.py 2>&1
```

`~/bin/analysis_python` 是预激活 analysis_env 的 python wrapper。

## 推荐工作流

### 第一步：数据探索（必须）

先选 `d` 列出 Job 表，了解有哪些 Job 及其模拟数量。如果需要看具体 Task 列表，再选对应 Job：

```bash
# 只列出 Job 表
printf "d\n" | ~/bin/analysis_python analysis_console.py 2>&1

# 选 Job 0 后列出所有 Task（用 head 截断避免输出过大）
printf "d\n0\nall\n" | ~/bin/analysis_python analysis_console.py 2>&1 | head -200
```

### 第二步：试运行（参数 ID 未知时）

参数 ID（温度、磁场、密度等）在不同数据选择中可能不同。不确定时用少量数据试运行，只填到参数选择器那一步，观察参数列表后 EOF 自然退出。

```bash
printf "d\n0\n0-2\nr\ny\n" | ~/bin/analysis_python analysis_console.py 2>&1
```

### 第三步：正式运行

根据前两步获得的信息，构造完整的 stdin 序列并执行。

## 交互层级地图

### 主循环命令
| 命令 | 功能 | 后续需要几层输入 |
|------|------|------------------|
| `d`  | 选择数据目录 | 2 层（Job 选择 → Task 选择）|
| `s`  | 选择分析模块 | 1 层（模块列表选择）|
| `r`  | 运行分析 | 见下方「r 运行时的交互层级」|
| `t`  | 工具 | 不确定层数，不建议通过管道操作 |
| `q`  | 退出 | 1 层（确认 y/n）|

### 选择器的输入格式（所有层级通用）
- `all` — 全选
- `0` — 选第 0 项
- `1,3,5` — 多选
- `1-5` — 范围选择
- `1-3,7,9-12` — 混合
- 空行（回车）— 接受默认值（通常是 `all`）

### `r` 运行时的交互层级（对比类模块）

`r` 命令会触发多层交互式 prompt，具体层数取决于数据。典型流程：

```
1. [y/n] 是否合并统计性重复模拟？默认 y → 输入 y
2. 参数选择器：列出变化参数（带 ID），选一个 → 输入参数 ID（如 2）
3. 操作选择：x=设为 X 轴 / f=筛选 → 输入 x
4. [可能重复 2-3] 如果还有剩余变化参数，会再次提示
5. 文件名后缀：回车跳过 → 输入空行
```

**注意**：参数 ID 在不同数据选择中可能不同。如果不确定，先用少量数据试运行一次，观察参数列表后再构造完整序列。

### stdin 序列构造示例

**场景 1：选目录 + 运行对比分析（含参数选择）**
```
d          ← 主菜单：选目录
0          ← Job 选择器：选 Job 0
0-11       ← Task 选择器：选 ID 0 到 11
r          ← 主菜单：运行分析
y          ← 合并重复模拟：是
2          ← 选择变化参数 ID（如 target_sigma）
x          ← 设为 X 轴
（空行）   ← 跳过文件名后缀
```
printf 格式: `printf "d\n0\n0-11\nr\ny\n2\nx\n\n"`

**场景 2：选目录 + 选模块 + 运行**
```
d          ← 主菜单：选目录
0          ← Job 选择器
all        ← Task 选择器：全选
s          ← 主菜单：选模块
6          ← 模块选择器：选模块 6
r          ← 运行（后续按模块类型填入交互）
```

- `plasma d all all r` — 选所有目录，运行
- `plasma 分析 bbn_sigma_scan` — 语义化描述，需要你自行推断 stdin 序列

## 注意事项

1. **timeout**: 分析运行可能耗时很长，设置足够的 timeout（建议 300000ms 起步，复杂分析可用 600000ms）
2. **输出可能很大**: 336 个 Task 的表格输出约 33KB，注意输出截断
3. **`r` 命令的交互层数不固定**: 依赖于数据中有多少变化参数、是否有重复模拟等。建议先小范围试运行以了解完整交互层级
4. **Job 中 count=0 的会被自动过滤**: 选了 `all` 也不会包含空 Job
5. **程序已修复 EOFError 处理**: stdin 耗尽时会干净退出，不会无限循环
6. **不要在序列末尾添加退出指令（q/y）**: 退出序列可能在交互层数不确定时误触其他 prompt，让程序通过 EOF 自然退出即可
7. **模块选择可跨会话持久化**: 程序启动时会恢复上次选择的模块（"已恢复 1 个模块选择"），无需每次都执行 `s` 命令
